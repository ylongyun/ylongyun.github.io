封装性的理解

面向对象的特征之一：封装与隐藏

封装性的实际应用

当我们创建一个类的对象以后，我们可以通过”对象.属性”的方式，对对象的属性进行赋值。这里，赋值操作要受到属性的数据类型和存储范围的制约。除此之外，没有其他制约条件。但是，在实际问题中，我们往往需要给属性赋值加入额外的限制条件。这个条件就不能在属性声明时体现，我们只能通过方法进行限制条件的添加。（比如：对类的属性年龄进行赋值就需要所赋的值大于零）同时，我们需要避免用户再使用”对象.属性”的方式对属性进行赋值（不这样做的话我们的设置的方法就没有什么什么卵用了┐(´-｀)┌），故需要将属性声明为私有的(private). 。
–>此时，针对于属性就体现了封装性。

封装性的体现

1.我们将类的属性xxx私有化(private),同时，提供公共的(public)方法来获取(getXxx)和设置(setXxx)此属性的值。
2.不对外暴露的私有的方法。
3.单列模式。

封装性的体现，需要权限修饰符来配合。

1.Java规定的4种权限（从小到大排列）：private、缺省、protected 、public 。
2.4种权限可以用来修饰类及类的内部结构：属性、方法、构造器、内部类。
3.特别的修饰类的话，只能使用：缺省、public。

初识栈

栈（stack）又名堆栈，它是一种运算受限的线性表。限定仅在表尾进行插入和删除操作的线性表。这一端被称为栈顶，相对地，把另一端称为栈底。向一个栈插入新元素又称作进栈、入栈或压栈，它是把新元素放到栈顶元素的上面，使之成为新的栈顶元素；从一个栈删除元素又称作出栈或退栈，它是把栈顶元素删除掉，使其相邻的元素成为新的栈顶元素。

刚开始学习栈时，我是充满了对新知识的好奇的。结果等我看完书上的描述，脑海之中就想到了链表，心想这东西和链表储存数据的形式是一样的呀，就是限定了对它的操作，只允许在表尾进行插入和删除节点，具体的操作和链表一摸一样，或者说比链表简单一点。白瞎了我的好奇，换汤不换药而已。因为之前写过对链表的操作了，所以今天就只写了对顺序栈的操作。如下：

1.定义顺序栈

typedef struct Stack
{
    int data[100];
    int top;
}LinkStack;

2.初始化栈

LinkStack *initStack(LinkStack *stack)//初始化栈 
{
    stack->top = -1;
    return stack;
}

3.入栈

void enterStack(LinkStack *stack)//入栈操作 
{
    if(stack->top+1>99)
        printf("栈已满");
    else
    {
        stack->top++;//栈顶后移 
        scanf("%d",&stack->data[stack->top]);//输入新成员元素 
    } 
}

4.出栈

int outStack(LinkStack *stack)//出栈操作 
{
    if(stack->top == -1)
    {
        printf("栈为空");
        return -1;
    }
    else
    {
        stack->top--;//栈顶前移
        return stack->data[stack->top+1];//返回新成员元素 
     } 
}

That is all.

约瑟夫环问题

约瑟夫环问题

问题描述:编号为1.2..n的n个人按顺时针方向围坐一圈,每人持有一个密码(正整数)。
开始时任选一个整数作为报数上界m,从第一个人开始顺时针自1开始顺序报数，报到m时停止报数。
报m的人出列，将他的密码作为新的m值,从他在顺时针方向上的下一个人开始重新从1报数，如此下去,
直至所有的人全部出列为止。试设计一个程序,求出出列顺序。

思路

在问题描述可以知道，在每一次报数中我们需要找到第m个成员，并读取他的密码作为下一次报数的m值，由于m值可能大于n(所有人的总数)，因此在一次报数中会出现成员重复的现象，因此采用无头结点的循环单链表的数据结构来存放数据。由于成员出列后就会将成员所在的节点删除，所以解决问题的关键就是找到要出列节点的前一个节点，从而删除出列的节点，然后读取出列节点的密码作为下一次报数的m值。循环上述步骤直到只剩下一个节点，然后将此节点出列，问题解决。

代码实现

#include <stdio.h>
#include <stdlib.h>
typedef struct Node
{
    int order;//成员序号
    int code;//成员密码
    struct Node* next;//下一个结点的地址

}LinkList;

LinkList *create (int length)//尾插法创建循环链表
{
    int i;
    LinkList *head,*node,*pre;
    head=(LinkList*)malloc(sizeof(LinkList));
    head->next=NULL;
    pre=head;
    for (i=1;i<=length;i++)
    {
        node=(LinkList*)malloc(sizeof(LinkList));
        node->order=i;//结点空间开辟顺序即为此节点的序号
        scanf("%d",&node->code);//输入成员密码
        pre->next=node;
        pre=node;
    }
    node->next=head->next;//去除头结点
    return head->next;//返回第一个结点的地址
}

int yuesefu(LinkList* head,int m)
{
    int i;
    LinkList* pre,* node;
    pre = head;
    while(head->next != head)//循环结束条件：链表中只剩一个结点
    {
        if(m != 1)//此时
        {
            for(i = 1;i < (m-1);i++)
            {
            pre = pre->next; 
            }
            node = pre->next;

        }
        else{
            pre = head->next;
            while(pre->next != head){
                pre = pre->next;
            }
            node = head;
        }
        printf("%d\t",node->order);    //输出成员序号
        pre->next = node->next;
        head = node->next;
        pre = head;
        m = node->code;//读取密码
    }
    printf("%d\n",head->order);//输出最后一个出列的成员序号
    return 0;
}

int main()
{
    LinkList* head;
    head=create(7);
    yuesefu(head,20);
    return 0;
}

单链表

数据结构-单链表

在数据结构的学习中，第一个学的就是单链表，虽然在第一个学期的c语言中已经接触到这个东西，但实际上对链表还是一片模糊。在之前的几天里，慢慢地又把链表给学习了一遍，将单链表的基础操作自己写了一遍。如下。

1.定义结构体

在我理解中，链表就是由若干个结构体链接而成的，结构体内存放我们的数据，因此在使用链表之前要先根据你的需求定义结构体。 

typedef struct Node 
{
   int data;
   struct Node* next;

}LinkList;

2.初始化链表（尾插法建立链表）

在链表中添加头节点会简化对链表的操作，头节点内不储存任何数据，只起到一个确定链表地址的作用。 

LinkList* creatList(int n)  
{
   LinkList* head;  
   if (n == 0)//n为0的即生成一个空链表  
   {  
       head = (LinkList*)malloc(sizeof(LinkList));
       head->next = NULL;//封闭节点
       head->data = NULL;

   }
   else
   {
       int i;
       LinkList* node, * last;
       head = (LinkList*)malloc(sizeof(LinkList));
       last = head;
       for (i = 0;i < n;i++)   
       {
           node = (LinkList*)malloc(sizeof(LinkList));//为新节点开辟空间
           scanf("%d", &node->data);
           last->next = node;//将新节点连接到链表上
           last = node;//使新节点成为最后一个节点
           node->next = NULL;//封闭最后一个节点
       }

    }
    return(head);
}

3.输出链表

链表的输出需要注意节点的更替，且要完整的输出，不能出现遗漏。 

void print(LinkList* head)
{
    LinkList* node;
    if (head->next == NULL)//判断是否为空链表
        printf("there are nothing in the list.");
    else
   {
    node = head->next;//从第一个结点开始输出
    do {
        printf("%d\n", node->data);//输出节点内容
        node = node->next;//切换下一个节点
       } while (node->next != NULL);//输出最后一个节点后终止循环
   }  
   printf("%d\n", node->data);  
/*这里需要注意，当while循环结束时，最后一个节点地内容并没有输出，由while的结束循环标志可知此时node  
指向最后一个节点，因此我们只需要将node->data输出就可以了。*/
}

4.1按值查找

此操作用于确定链表中是否有某个元素。

{
    LinkList* node;

4.2按顺序查找

此操作用于确定链表中某一个元素的位置 

int find_order(LinkList* head, int num)  
{
    int i = 1;
    LinkList* node = head->next;
    while (node->data != num) //判断该节点数据是否为我们所查找的元素
    {
        i++;
        node = node->next;//切换节点
    }
    return(i);//返回所查找的元素的位置顺序
}

5.插入节点

更改链表的内容，在给定的序号后插入。 

void add(LinkList* head, int num)
{
    LinkList* pre,* node;
    int i=0;
    node = (LinkList*)malloc(sizeof(LinkList));//为新节点开辟空间
    pre = head;
    scanf("%d", &node->data);
    for (;i < num;i++) //确定插入位置前一个节点
    {
        pre = pre->next;
    }
    node->next = pre->next;//将插入位置之后的节点连接到新节点的后面
    pre->next = node;//将新节点连接到插入位置前一个节点的后面
}

6.删除节点

删除特定序号的节点，并释放该节点所占用的空间。 

void delete(LinkList* head, int num)
{
    LinkList* pre, * cur, * node;
    int i = 0;
    pre = head;
    for (;i < num-1;i++)
    {
        pre = pre->next;//确定被删除节点的前一个结点的位置
    }
    node = pre->next;//找到被删除节点的位置
    cur = node->next;//找到被删除节点的后一个节点的位置
    pre->next = cur;//将被删除的节点从链表中移除
    free(node);//释放被删除节点的内存

}

7.测量链表的长度

链表的长度不包括头节点，函数返回值为链表的长度。

}
