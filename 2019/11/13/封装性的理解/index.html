封装性的理解

  • Java

面向对象的特征之一：封装与隐藏

封装性的实际应用

当我们创建一个类的对象以后，我们可以通过”对象.属性”的方式，对对象的属性进行赋值。这里，赋值操作要受到属性的数据类型和存储范围的制约。除此之外，没有其他制约条件。但是，在实际问题中，我们往往需要给属性赋值加入额外的限制条件。这个条件就不能在属性声明时体现，我们只能通过方法进行限制条件的添加。（比如：对类的属性年龄进行赋值就需要所赋的值大于零）同时，我们需要避免用户再使用”对象.属性”的方式对属性进行赋值（不这样做的话我们的设置的方法就没有什么什么卵用了┐(´-｀)┌），故需要将属性声明为私有的(private). 。
–>此时，针对于属性就体现了封装性。

封装性的体现

1.我们将类的属性xxx私有化(private),同时，提供公共的(public)方法来获取(getXxx)和设置(setXxx)此属性的值。
2.不对外暴露的私有的方法。
3.单列模式。

封装性的体现，需要权限修饰符来配合。

1.Java规定的4种权限（从小到大排列）：private、缺省、protected 、public 。
2.4种权限可以用来修饰类及类的内部结构：属性、方法、构造器、内部类。
3.特别的修饰类的话，只能使用：缺省、public。
